Section 04

Key Concepts

CLAUDE.md — Project Memory

A markdown file in your project root that Claude reads automatically. Document coding standards, architecture, and testing instructions for persistent cross-session context.

Slash Commands

Built-ins: /help, /compact, /cost, /model, /status, /bug. Create custom commands as markdown files in .claude/commands/.

Subagents — Delegation

Claude spawns sub-agents with clean context windows for focused tasks (explore, plan, implement). Returns only summaries — prevents context bloat and keeps costs down.

Permission System

Claude asks before impactful actions. Configure fine-grained allow/block rules in .claude/settings.json.

Section 05

Extensibility — Hooks, MCP & Skills

🪝 Hooks

Shell commands that fire at lifecycle events (pre-commit, post-tool-use, etc.). Use for linting, formatting, and security checks — deterministically, every time.

🔌 MCP

Open standard for connecting Claude to external services — GitHub, Sentry, Slack, databases, and 300+ others. Donated to the Linux Foundation (Agentic AI Foundation).

🎯 Skills

Organized folders of instructions, scripts, and resources that Claude loads dynamically. Custom slash commands on steroids — reusable expertise packages.

Browse & install MCP servers: registry.modelcontextprotocol.io  ·  Community plugins: awesome-claude-code on GitHub

Section 06

CI/CD & Automation

Claude Code integrates with GitHub Actions and GitLab CI/CD pipelines via the official anthropics/claude-code-action.

  • Trigger Claude by mentioning @claude in PR comments
  • Automate code reviews on every PR
  • Turn issues into working code & PRs automatically
  • Translate strings, update docs, fix lint errors
  • Headless mode: claude -p for scripting
Setup in 3 Steps
1
Run /install-github-app inside Claude Code
2
Add your API key as a GitHub Actions secret
3
Add a workflow YAML file referencing the action
