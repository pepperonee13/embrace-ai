01 — What is Claude Code

An AI Coding Agent

Originally launched as a terminal-based CLI, Claude Code is now accessible through multiple interfaces:

  • CLI agent — claude command in your terminal
  • Claude web app — browser-based access
  • Integrations — automation workflows

Think of it as:
A junior developer that can operate tools, not just generate text.

What it can do
  • Read files
  • Write & refactor code
  • Run commands & tests
  • Analyze repositories
  • Create commits & PRs
01 — What is Claude Code

Install & Workflow

Install globally via npm:

npm install -g @anthropic-ai/claude-code

Then start inside any project:

claude
Simplified loop

You describe a task
↓ Claude explores the repo
↓ Claude performs actions (tools)
↓ You review results
↓ Iterate

03 — Claude Model Lineup

Three Families, One Right Tool

Model Version Personality Best For
Haiku Fast 4.5 Very fast, inexpensive Automation, tooling, simple tasks
Sonnet Daily driver 4.6 Balanced, near-Opus coding ability Daily coding tasks
Opus Flagship 4.6 Highest reasoning, 1M token context Complex architecture & debugging

💡 Note: Sonnet 4.6 now outperforms previous-generation Opus on many coding benchmarks (94% computer use accuracy) — making it the daily-driver for most developers.

03 — Claude Model Lineup

Switching Models in the CLI

Switch model at any point during a session:

/model claude-opus-4-6

Typical per-model workflow:

Haiku 4.5

Automation scripts, quick lookups, low-cost tooling

Sonnet 4.6

Daily coding tasks — handles 90%+ of work at lower cost

Opus 4.6

Hard reasoning, 1M token context, complex architecture

Rule of thumb: Start with Sonnet 4.6. Switch to Opus 4.6 when the problem gets hard.

05 — Tokens, Limits & Pricing

Understanding Tokens

A token is roughly ¾ of a word. Everything you send and receive counts:

  • Input prompts
  • Files read by Claude
  • Generated responses
  • Tool outputs
Content Approx. tokens
Short prompt ~20
Long function ~200
Medium source file 1,000+
Large codebase context 10,000+
Practical tips to reduce usage

Clear old context often  ·  Use focused prompts  ·  Reference specific files  ·  Avoid long conversation chains

06 — Customization & Automation

CLAUDE.md — AI Onboarding

Claude reads this file automatically at repo root. Think of it as team onboarding instructions for your AI colleague.

  • Consistent behavior across sessions
  • Shared team conventions
  • Reusable automation rules
# Project Rules

Always run tests before committing.
Use TypeScript strict mode.

Branch naming:
feature/[ticket-id]-short-description

Never edit files in /generated
06 — Customization & Automation

Custom Slash Commands

Define reusable prompts as files in:

.claude/commands/

Invoke with:

/review

Commands can be shared across the team via the repo.

Example: .claude/commands/review.md

Review the staged changes for bugs
and security issues only.

Ignore formatting and style comments.
06 — Customization & Automation

Hooks — Automatic Actions

Hooks run shell commands automatically during agent activity.

  • Automatic linting
  • Auto-formatting on write
  • Test execution
  • Policy enforcement

Example: run Prettier after every TypeScript write

{
  "hooks": {
    "PostToolUse": [{
      "matcher": "Write(*.ts)",
      "hooks": [{
        "type": "command",
        "command": "prettier --write $file"
      }]
    }]
  }
}
07 — How Claude Code Works

Tool-Based Agent Architecture

Claude doesn't directly edit files — it calls tools, reads the output, then decides the next step.

Tool Purpose
ReadFileInspect source code
WriteFileModify code
RunCommandExecute shell commands
SearchExplore repository
Git toolsCommit, diff, PR workflows
Autonomous loop

User task
↓ Claude plans actions
↓ Claude uses tools
↓ Results returned
↓ Claude decides next step
↓ Repeat until done

07 — How Claude Code Works

Model Context Protocol (MCP)

Claude can interact with external systems via MCP servers — safely and without custom integrations.

🐙 GitHub

Query repositories, issues, PRs, and actions directly.

🗄️ Databases

Read schemas, run queries, inspect data safely.

🔌 Internal APIs

Documentation systems, internal tooling, custom services.

💡 MCP allows Claude to query external systems without giving it unrestricted access — each server exposes only what it chooses to.
