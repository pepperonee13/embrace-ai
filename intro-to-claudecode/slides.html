01 — What is Claude Code?

An AI Coding Agent in Your Terminal

  • A CLI tool by Anthropic — not a plugin, not a sidebar, not a chat box
  • Can read files, write code, run tests, make commits, and push PRs
  • Powered by Anthropic's Claude models under the hood
  • Think: a dev who never sleeps, never complains about legacy code, and actually reads the docs
Install in one line
npm install -g @anthropic-ai/claude-code

Then run claude in your project directory. That's it.

How it works

You describe what you want
↓
Claude reads your codebase
↓
Claude acts
↓
You review & repeat

05 — Tokens & Usage Limits

The Battery Gauge You Need to Watch

What is a token?

Roughly ¾ of a word. A 200-line file ≈ a few thousand tokens. Every file Claude reads, every line it writes, every tool result — all count.

Track your spend
/cost    # token spend for this session
/usage   # remaining quota + reset timer

Usage resets on a 5-hour rolling window — shared across claude.ai, Claude Code, and Claude Desktop.

Plan~Tokens / 5hOpus
FreeLimited❌
Pro (~$20/mo)~44,000❌
Max 5× (~$100/mo)~88,000✅
Max 20× (~$200/mo)~220,000✅

API users get separate pay-per-token billing — no rolling windows.

05 — Tokens & Usage Limits

Token-Saving Tips That Actually Work

🧹
Use /clear often — don't drag stale context into new tasks
📄
Write a lean CLAUDE.md — Claude reads it every session, so keep it tight
🎯
Be specific: "fix the login bug in UserController.java" beats "fix the login stuff"
📦
Big diffs in one prompt > many small "refine this" follow-ups
Weekly caps (Aug 2025)

Weekly usage caps were introduced — but they only affect <2% of users with heavy Sonnet usage.

Unless you're using Claude Code all day every day, you won't hit them.

06 — Customization & Automation

CLAUDE.md — Your Project's Brain

A markdown file at the root of your project (or ~/.claude/CLAUDE.md globally) that Claude reads at the start of every session.

Think of it as onboarding docs for your AI colleague. Commit it to git — your whole team benefits.

Most impactful 15 minutes

Setting up a good CLAUDE.md is the highest-leverage thing you can do for your AI workflow.

# My Project

- Always use TypeScript strict mode
- Run `npm test` before committing
- Branch naming:
  feature/[ticket-id]-short-description
- Never edit /generated — auto-generated
06 — Customization & Automation

Custom Slash Commands — Macros for Your Workflow

Create repeatable workflows by dropping markdown files in .claude/commands/:

mkdir -p .claude/commands

Then invoke from any session:

/review

Or with parameters:

/fix-issue 1337
Example: .claude/commands/review.md
Review the staged changes for bugs
and security issues only.

Be concise. No style comments.
Why it matters

Turns your most-used prompts into a one-word command. Build a team library and everyone benefits.

06 — Customization & Automation

Hooks — The "Always Do This" Layer

Hooks run deterministic shell commands at specific lifecycle points. Unlike CLAUDE.md suggestions, hooks are rules that always fire.

Available hook events
  • UserPromptSubmit
  • PreToolUse
  • PostToolUse
  • Stop

Configure via the interactive /hooks command — no JSON editing required.

Example: Auto-run Prettier after every file write
{
  "hooks": {
    "PostToolUse": [{
      "matcher": "Write(*.ts)",
      "hooks": [{
        "type": "command",
        "command": "prettier --write $file"
      }]
    }]
  }
}
06 — Customization & Automation

Plugins — Share It All as a Bundle

Plugins package slash commands + hooks + MCP servers into one installable unit. Great for team-wide consistency. 

/plugin install <plugin-name>
What's bundled
  • Custom slash commands
  • Hook configurations
  • MCP server definitions
Best use case

Define your team's standards once, distribute as a plugin. New devs get the full setup instantly.
